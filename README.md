# Chapter Splitter Sigil Plugin

`Chapter Splitter Sigil` 是一个Sigil拆章插件，用来把一整份 TXT 小说或长文按标题规则拆成多个 XHTML 章节，并直接导入当前电子书。

## 推荐使用流程

建议按这个顺序操作：

1. 选择本地 TXT 文件
2. 打开 `txt预处理`，按需要勾选去空行、去段前缩进
3. 点击 `自动分析`，让插件先填入常见规则
4. 必要时手动补充或修改正则规则、标题级别、是否分割
5. 点击 `预览`，检查层级、标题、忽略项和可疑问题
6. 运行 `断章检查`，确认是否存在跳章或编号异常
7. 调整模板和文件命名规则
8. 点击 `执行分章`

## 主要功能

- 支持中文、英文常见章节标题识别
- 支持 `h1` 到 `h6` 多级标题
- 支持自动分析并生成常见规则
- 支持预览章节树、搜索标题、按级别筛选
- 支持双击编辑预览中的标题
- 支持修改预览中的标题级别
- 支持勾选忽略章节，使其不参与导出
- 支持断章检查，发现跳章和编号异常
- 支持长标题高亮提示
- 支持自定义 XHTML 模板
- 支持自定义文件命名规则
- 支持中英文界面

## 关于标题级别

- 每条规则都可以指定 `h1` 到 `h6`
- 匹配成功后，标题会按所选级别包裹成对应的 HTML 标签
- 通常可以这样理解：

- `h1` 适合卷、部、篇这类大层级
- `h2` 适合章、回这类主层级
- `h3` 适合节、小节这类次层级

标题级别不仅影响生成的标签，也会影响预览树结构、命名规则中的层级变量，以及断章检查的分组方式。

## 关于“分割”

- 每条规则都有一个“分割”勾选框
- 勾选时，命中该规则会创建新的章节文件
- 不勾选时，该行仍会被套上对应的 `h1` 到 `h6` 标签，但不会在这里切成独立页面

## 关于规则顺序

- 规则按界面中的先后顺序执行，也就是从上到下
- 当多条规则都可能命中同一行时，排在前面的规则会先起作用
- 如果原文里存在相似标题，通常应该把“更具体”的规则放在前面，把“更宽泛”的规则放在后面

例如：

- `^第.+卷 第.+章.*$`
- `^第.+卷.*$`

这类组合里，通常应该先写前者，再写后者。否则更宽泛的规则可能先把内容吃掉，导致层级判断不符合预期。

## 关于自动分析

- 自动分析会根据当前启用的预设规则，尝试从文本中找出常见标题模式
- 它更适合“多数标题格式较统一”的文本，例如一般的网文小说

当前版本默认最多支持 15 条规则。

## 关于预览

在预览里可以：

- 查看章节树和层级关系
- 查看每章标题
- 查看章节正文首段
- 搜索标题关键字
- 按级别筛选
- 只看有问题的章节
- 控制是否显示已忽略章节
- 双击编辑标题
- 在级别列里修改标题级别
- 勾选“忽略”让章节不参与导出
- 右键复制标题文本

如果预览没有任何结果，通常说明当前规则没有匹配到有效标题，或者文本本身还需要先做预处理。

## 关于忽略章节

- 预览中的“忽略”列勾选后，该章节不会参与最终导出
- 被忽略的章节会以删除线样式显示，便于和正常章节区分
- 这适合排除误识别标题、空章节、重复章节或不想导出的内容

## 关于断章检查

- 断章检查会提取标题中的第一串可识别数字进行比对
- 支持阿拉伯数字、全角数字、常见中文数字
- 不同标题级别会分别检查，不会互相干扰

例如这些数字形式都可以参与检查：

- `2002`
- `２００２`
- `二〇〇二`
- `两千零二`

如果前后章节编号不是正常递增，预览中会给出警示，报告窗口也可以帮助定位问题章节。

## 关于长标题提示

- 预览支持“标题高亮阈值”
- 当标题长度超过阈值时，会在预览中高亮提示

## 关于模板

模板本质上是一段 XHTML 页面代码，至少需要保留两个占位符：

- `[TITLE]`
- `[MAIN]`

含义如下：

- `[TITLE]` 用于章节标题文本
- `[MAIN]` 用于章节正文内容，包含标题标签和正文段落

如果删掉其中任意一个，输出内容就会不完整。

## 关于文件命名规则

当前版本按标题级别分别设置命名规则，默认可配置 `h1`、`h2`、`h3`。

可用占位符包括：

- `{000}`：全局序号，3 位补零
- `{$$$}`：同级相对序号，3 位补零，遇到上级变化会重置
- `{title}`：当前标题，特殊字符会被替换为下划线
- `{h1_no}`：当前 `h1` 序号
- `{h2_no}`：当前 `h2` 序号
- `{h1_no3}`：3 位补零的 `h1` 序号
- `{h2_no3}`：3 位补零的 `h2` 序号
- `{h1}`：最近一个 `h1` 标题文本
- `{h2}`：最近一个 `h2` 标题文本

默认命名示例：

- `Vol001_Chapter000.xhtml`
- `Vol001_Chapter003.xhtml`

如果生成的文件名重复，插件会自动追加后缀避让，不会直接覆盖同名文件。
